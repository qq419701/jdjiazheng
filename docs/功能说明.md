# 功能说明文档

## 一、供货商（Vendor）账号体系

### 1.1 供货商角色介绍

供货商是一种特殊的后台账号角色（`vendor`），专为卡密批次供应商设计：
- 供货商登录后**只能看到「订单中心」**，其他所有菜单（卡密工作台、套餐管理、数据看板、业务设置、地区管理、子账号管理等）全部隐藏
- 供货商通过绑定的**批次ID列表**实现数据隔离，只能查看该批次下卡密对应的订单
- 数据隔离在后端实现，前端仅做菜单隐藏，保障安全性

### 1.2 如何创建供货商账号

1. 使用 `super` 超级管理员账号登录后台
2. 进入「子账号管理」页面
3. 点击「新增子账号」按钮
4. 填写用户名、密码、昵称（可选）
5. 角色选择「供货商（按批次授权）」
6. 在「可见业务」中勾选该供货商可以看到的业务类型（不选则显示所有）
7. 在「绑定批次」中勾选该供货商对应的卡密批次
8. 点击「创建」完成

### 1.3 如何绑定批次

- 批次在「卡密工作台」中生成卡密时自动创建
- 每次生成卡密会产生一个批次记录（含批次号、业务类型、服务类型等）
- 在子账号管理的编辑弹窗中，选择供货商角色后会显示所有批次列表（按业务类型分组）
- 勾选对应批次即可完成绑定
- 绑定后供货商只能看到该批次下卡密使用产生的订单

### 1.4 供货商登录后的功能范围

- ✅ 订单中心（家政/洗衣/充值/三角洲，仅显示绑定批次对应的订单）
- ✅ 手动添加订单（可以手动补单）
- ❌ 数据看板
- ❌ 卡密工作台
- ❌ 套餐管理
- ❌ 业务设置
- ❌ 地区管理
- ❌ 子账号管理
- ❌ SUP日志

## 二、手动新建订单功能

### 2.1 功能位置

在「订单中心」页面的每个业务 Tab（家政/洗衣/充值/三角洲）的搜索栏最后，有一个绿色「+ 添加订单」按钮。

### 2.2 操作步骤

1. 进入「订单中心」，切换到对应业务 Tab
2. 点击搜索栏末尾的「+ 添加订单」按钮
3. 在弹窗中：
   a. 选择套餐（从下拉列表中选择对应套餐）
   b. 系统自动显示将要使用的卡密及剩余库存数量
   c. 如需手动指定卡密，点击「手动指定卡密」展开输入框
   d. 填写客户信息（姓名、手机号、地址等，根据业务类型不同字段不同）
   e. 填写备注（选填）
4. 点击「确认创建」完成
5. 成功后自动刷新订单列表，并显示生成的订单号

### 2.3 卡密自动分配机制

- 选择套餐后，系统自动查找该套餐下**最早生成的未使用卡密**（FIFO 先进先出）
- 使用数据库事务 + 行锁防止两个管理员同时创建订单时抢用同一张卡密
- 若套餐下没有可用卡密，系统提示「当前套餐无可用卡密，请先生成卡密」
- 也可手动输入指定卡密号（点击「手动指定卡密」展开输入框）

## 三、数据库变更说明

### 3.1 新增字段清单

#### `admins` 表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `role` ENUM | 新增 `vendor` 值 | 角色枚举新增供货商类型 |
| `vendor_batch_ids` | TEXT | 供货商绑定的批次ID列表（JSON数组） |
| `vendor_business_types` | VARCHAR(100) | 供货商可见的业务类型（逗号分隔） |

#### `card_batches` 表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `vendor_id` | INTEGER | 所属供货商admin_id（NULL=无供货商） |

#### `products` 表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `vendor_id` | INTEGER | 关联的供货商admin_id（NULL=无供货商） |

#### `orders` 表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `order_source` | VARCHAR(20) | 订单来源：customer=客户H5 manual=管理员手动创建 |

### 3.2 运维提示

**若项目使用 `sequelize.sync({ alter: true })` 或 `force: false`：**
- 新字段将在服务重启后自动添加到数据库，无需手动执行 ALTER TABLE

**若需要手动迁移（生产环境推荐）：**

```sql
-- admins 表：修改 role ENUM
ALTER TABLE admins MODIFY COLUMN role ENUM('super','admin','sub','vendor') DEFAULT 'admin' COMMENT '角色';

-- admins 表：新增供货商字段
ALTER TABLE admins
  ADD COLUMN vendor_batch_ids TEXT NULL COMMENT '供货商绑定的批次ID列表（JSON数组格式）',
  ADD COLUMN vendor_business_types VARCHAR(100) NULL COMMENT '供货商可见的业务类型（逗号分隔）';

-- card_batches 表：新增供货商关联
ALTER TABLE card_batches
  ADD COLUMN vendor_id INT NULL COMMENT '所属供货商admin_id';

-- products 表：新增供货商关联
ALTER TABLE products
  ADD COLUMN vendor_id INT NULL COMMENT '关联的供货商admin_id';

-- orders 表：新增订单来源
ALTER TABLE orders
  ADD COLUMN order_source VARCHAR(20) DEFAULT 'customer' COMMENT '订单来源：customer客户H5 manual管理员手动创建';
```
