# 京东家政预约代下单系统 - 宝塔部署教程

## 一、服务器环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | CentOS 7.x / Ubuntu 18.04+ |
| CPU | 2核以上 |
| 内存 | 4GB以上 |
| 硬盘 | 40GB以上 |
| 带宽 | 5Mbps以上 |

---

## 二、安装宝塔面板

### 2.1 CentOS系统安装命令
```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

### 2.2 Ubuntu系统安装命令
```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

### 2.3 安装完成后
安装完成后会显示：
- 面板地址：`http://服务器IP:8888/xxxx`
- 用户名：随机生成
- 密码：随机生成

**重要：** 请记录面板地址和账号密码，首次登录后立即修改密码。

## 三、安装必要软件

登录宝塔面板后，点击左侧 **软件商店**：

1. 安装 **Nginx**（最新版本）
2. 安装 **MySQL 5.7**
3. 安装 **Node.js 版本管理器**（PM2管理器）
   - 安装完成后，在PM2管理器中安装 Node.js 18.x 或 20.x

---

1. 宝塔面板左侧点击 **数据库**
2. 点击 **添加数据库**
3. 填写：
   - 数据库名：`jdjiazheng`
   - 用户名：`jdjiazheng`（自动生成）
   - 密码：设置强密码（**记录备用**）
4. 点击提交

## 六、配置环境变量

1. 进入后端目录：
```bash
cd /www/wwwroot/jdjiazheng/backend
```

2. 复制并编辑环境变量文件：
```bash
cp .env.example .env
```

3. 用宝塔文件管理器编辑 `.env` 文件，填写正确的配置：
```env
# 数据库配置（填写刚才创建的数据库信息）
DB_HOST=localhost
DB_PORT=3306
DB_NAME=jdjiazheng
DB_USER=jdjiazheng
DB_PASSWORD=你的数据库密码

# JWT密钥（建议修改为随机字符串）
JWT_SECRET=jdjiazheng_secret_2024_xxxxxxxxx

# 服务端口
PORT=5500

# 管理员初始密码
ADMIN_DEFAULT_PASSWORD=admin123

# Puppeteer配置
PUPPETEER_HEADLESS=true
PUPPETEER_TIMEOUT=30000

# 环境
NODE_ENV=production
```

## 十、PM2启动后端服务

1. 宝塔面板 → **软件商店** → 找到 **PM2管理器** → 点击管理
2. 点击 **添加项目**
3. 填写：

   - 项目名称：`jdjiazheng`
   - 运行目录：`/www/wwwroot/jdjiazheng/backend`
4. 点击确定启动

### 或通过命令行启动：
```bash
cd /www/wwwroot/jdjiazheng/backend
pm2 start app.js --name jdjiazheng
pm2 save          # 保存PM2配置
pm2 startup       # 设置开机自启
```

验证服务是否启动：
```bash
pm2 status
curl http://localhost:5500/api/settings/public
```

## 十一、配置Nginx反向代理

### 11.1 创建网站
1. 宝塔面板 → **网站** → **添加站点**
2. 填写域名（如：`jdjiazheng.example.com`）
3. PHP版本选择 **纯静态**
4. 提交创建

### 11.2 配置反向代理
1. 点击站点 → **反向代理** → **添加反向代理**
2. 填写：
   - 代理名称：`jdjiazheng`
   - 目标URL：`http://127.0.0.1:5500`
3. 点击提交

### 11.3 或手动配置Nginx

点击站点 → **配置文件**，将内容替换为：

```nginx
server {
    listen 80;
    server_name 你的域名;

    # 前端H5
    location / {
        proxy_pass http://127.0.0.1:5500;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
    }
}
```

## 十四、生产环境安全配置

### 14.1 修改默认密码
登录后台 → 没有密码修改功能时，可直接在数据库修改：
```sql
-- 在宝塔MySQL管理中执行
USE jdjiazheng;
UPDATE admins SET password = '$2b$10$新的bcrypt密码哈希' WHERE username = 'admin';
```

或重新初始化时修改 `.env` 中的 `ADMIN_DEFAULT_PASSWORD`。

### 14.2 修改JWT密钥
修改 `.env` 中的 `JWT_SECRET` 为随机强密钥：
```bash
# 生成随机密钥
openssl rand -base64 32
```

### 14.3 开启宝塔防火墙
- 只开放必要端口：80、443、22（SSH）
- 关闭5500端口（通过Nginx代理，无需直接暴露）

## 十五、常见问题排查

### Q1: 数据库连接失败
```
❌ 数据库连接失败: Access denied for user
```
**解决：** 检查 `.env` 中的数据库密码是否正确，宝塔数据库密码和.env中的密码保持一致。

### Q2: npm install 超时
```
npm ERR! network timeout
```
**解决：** 切换npm镜像源
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q3: Puppeteer启动失败
```
Error: Failed to launch the browser process
```
**解决：** 安装Chromium依赖
```bash
# CentOS
yum install -y chromium chromium-headless

# Ubuntu
apt-get install -y chromium-browser
```

### Q4: 端口5500被占用
```
Error: listen EADDRINUSE :::5500
```
**解决：** 查找并关闭占用进程
```bash
lsof -i:5500
kill -9 进程ID
```

### Q5: 前端页面404
**解决：** 确认前端已构建（`npm run build`），且后端app.js中静态文件路径正确。

### Q6: API请求跨域错误
**解决：** 检查Nginx配置是否正确反向代理到5500端口，后端已配置cors允许所有来源。

## 十六、更新部署流程

每次代码更新后执行：
```bash
cd /www/wwwroot/jdjiazheng

# 拉取最新代码
git pull

# 重新安装依赖（如有新依赖）
cd backend && npm install

# 重新构建前端
cd ../frontend-h5 && npm run build
cd ../frontend-admin && npm run build

# 重启后端服务
pm2 restart jdjiazheng
```
