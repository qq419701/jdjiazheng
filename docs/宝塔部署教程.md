# 宝塔面板部署教程（极简版）

## 访问地址说明

部署完成后，所有访问都通过同一个端口（5500），统一由Nginx转发：

| 用途 | 地址 |
|------|------|
| 客户预约页面 | `http://你的域名/{卡密}` |
| 后台管理 | `http://你的域名/admin` |

---

## 第一步：安装必要软件

登录宝塔面板 → 左侧菜单「软件商店」，依次安装以下软件：

1. **Node.js版本管理器** → 安装完后，进入管理 → 安装 Node.js **18.x 版本**
2. **MySQL 5.7**（安装时记住root密码）
3. **PM2管理器**（用于保持后端持续运行）
4. **Nginx**（用于域名反向代理）

---

## 第二步：创建数据库

宝塔面板 → 左侧「数据库」→「添加数据库」：

- 数据库名：`jdjiazheng`
- 用户名：`jdjiazheng`
- 密码：自己设置（**记住这个密码，后面要用**）
- 点击「提交」

---

## 第三步：上传代码到服务器

**方法一（推荐）：SSH命令行**

宝塔面板 → 终端 或 用SSH工具连接服务器，执行：

```bash
cd /www/wwwroot
git clone https://github.com/qq419701/jdjiazheng.git
```

**方法二：手动上传**

1. 把代码下载到本地（GitHub页面 → Code → Download ZIP）
2. 宝塔面板 → 文件 → `/www/wwwroot/` → 上传ZIP → 解压

---

## 第四步：配置环境变量

```bash
# 进入后端目录
cd /www/wwwroot/jdjiazheng/backend

# 复制配置模板
cp .env.example .env
```

用宝塔文件管理器打开 `/www/wwwroot/jdjiazheng/backend/.env`，修改以下内容：

```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=jdjiazheng
DB_USER=jdjiazheng
DB_PASS=你在第二步设置的数据库密码
JWT_SECRET=随便填一串字符比如abc123xyz456
PORT=5500
NODE_ENV=production
```

保存文件。

---

## 第五步：安装依赖 + 初始化数据库

在宝塔终端执行：

```bash
cd /www/wwwroot/jdjiazheng/backend
npm install
node init/initDB.js
```

看到输出 `✅ 数据库初始化完成` 即成功。

> 初始化后会自动创建默认管理员账号：**admin / admin123**

---

## 第六步：PM2启动后端服务

宝塔面板 → 左侧「PM2管理器」→「添加项目」：

| 配置项 | 填写内容 |
|--------|----------|
| 启动文件 | `/www/wwwroot/jdjiazheng/backend/app.js` |
| 运行目录 | `/www/wwwroot/jdjiazheng/backend` |
| 项目名称 | `jdjiazheng` |

点击「确定」，状态变为「运行中」即可。

---

## 第七步：Nginx反向代理

宝塔面板 → 「网站」→「添加站点」：

- 域名：填写你的域名（如 `jd.yourdomain.com`）
- 根目录随便（因为我们用反向代理，不用静态目录）
- PHP版本：纯静态

添加完后，点击域名进入「设置」→「反向代理」→「添加反向代理」：

| 配置项 | 填写内容 |
|--------|----------|
| 代理名称 | `jdjiazheng` |
| 目标URL | `http://127.0.0.1:5500` |

点击「提交」，开启反向代理。

---

## 第八步：申请SSL证书（HTTPS，推荐）

宝塔面板 → 网站 → 点击域名 → SSL → Let's Encrypt → 申请免费证书

申请成功后开启「强制HTTPS」。

---

## 验证测试

1. 打开浏览器访问 `http://你的域名/admin`
2. 输入账号 `admin`，密码 `admin123`
3. 登录成功 → 去「卡密管理」→「批量生成」→ 生成一个卡密（如 `ABC123XYZ`）
4. 新标签页访问 `http://你的域名/ABC123XYZ`
5. 看到京东家政预约页面 = 部署成功！ 🎉

## 以后修改代码怎么办？

**如果你直接在GitHub上改了代码（不涉及前端Vue文件）：**
```bash
cd /www/wwwroot/jdjiazheng/backend
git pull
pm2 restart jdjiazheng
```

**如果修改了前端H5代码（frontend-h5目录）：**
```bash
cd /www/wwwroot/jdjiazheng/frontend-h5
npm install
npm run build
pm2 restart jdjiazheng
```

**如果修改了后台管理代码（frontend-admin目录）：**
```bash
cd /www/wwwroot/jdjiazheng/frontend-admin
npm install
npm run build
pm2 restart jdjiazheng
```

> 构建完成后，文件自动输出到 `backend/public/` 目录，重启PM2即生效。

## 常见问题

**Q：访问域名显示404或白屏？**
A：检查PM2是否运行中；检查Nginx反向代理是否指向 `127.0.0.1:5500`

**Q：数据库连接失败，后台无法登录？**
A：检查 `.env` 文件里的 `DB_PASS` 是否正确；确认MySQL服务是否运行

**Q：卡密页面打开是空白页或加载提示？**
A：需要执行前端构建：`cd frontend-h5 && npm install && npm run build`

**Q：端口5500无法访问？**
A：使用Nginx反向代理时，5500端口无需对外开放，只需确保Nginx的80/443端口正常访问即可。如果不使用Nginx（直接访问5500），才需要在宝塔安全和服务器安全组中放行5500端口

**Q：PM2重启后服务没恢复？**
A：在宝塔PM2管理器里设置「开机自启」

## 目录结构说明

```
jdjiazheng/
├── backend/              ← 后端Node.js（主要在这里运行）
│   ├── app.js            ← 程序入口，端口5500
│   ├── .env              ← 环境变量（你需要配置这个）
│   ├── public/
│   │   ├── h5/           ← H5前端构建产物（已内置）
│   │   └── admin/        ← 后台管理构建产物（已内置）
│   └── init/initDB.js    ← 数据库初始化脚本
├── frontend-h5/          ← H5前端源码（需要时可修改后重新构建）
├── frontend-admin/       ← 后台管理源码（需要时可修改后重新构建）
└── docs/                 ← 文档目录
```
