# 京东家政预约代下单系统 - 需求文档

## 一、项目背景与目标

### 项目背景
随着京东家政服务的普及，越来越多的用户通过京东平台预约家政服务。本系统旨在为商家提供一套完整的京东家政预约代下单工具，通过卡密分发的方式，让客户自助完成预约信息填写，再由系统自动或人工代为在京东平台完成下单操作。

### 项目目标
1. **客户端**：提供简洁的H5预约页面，客户通过卡密链接访问，填写地址和预约时间
2. **管理端**：提供完整的后台管理系统，支持卡密生成、订单管理、京东账号管理
3. **自动化**：通过Puppeteer自动化操作京东平台，减少人工干预
4. **灵活配置**：支持按城市/地区自定义预约时间规则，适应不同区域的运营需求

---

## 二、用户角色

### 2.1 客户（C端用户）
- 通过卡密链接（`/link/{卡密}`）访问H5预约页面
- 填写服务地址（姓名、手机号、省市区、详细地址）
- 选择上门预约时间
- 提交预约申请

### 2.2 管理员（B端用户）
- 登录后台管理系统（`/admin`）
- 管理卡密（批量生成、查看、导出）
- 管理订单（查看详情、触发下单、更新状态）
- 管理京东账号（添加账号、自动登录、Cookie维护）
- 配置时间规则（按城市/地区设置约满天数）
- 系统设置（Banner、服务内容、下单须知等）

## 四、业务流程图

### 4.1 客户预约流程

```
客户收到卡密（来自商家/阿奇所发货软件）
         │
         ▼
访问 /link/{卡密}
         │
         ▼
系统验证卡密有效性
    ┌────┴────┐
    │ 无效    │ 有效
    ▼         ▼
显示无效页   显示预约主页
             │
             ▼
点击"服务信息"→ 填写地址页
（姓名/手机号/省市区/详细地址）
             │
             ▼
保存地址后 → 选择上门时间
（系统根据城市返回时间表）
             │
             ▼
点击"立即预约" → 确认订单页
             │
             ▼
确认提交 → 后端创建订单
             │
             ▼
卡密标记为已使用
             │
             ▼
显示预约成功页（订单号+信息）
```

### 4.2 管理员下单流程

```
管理员登录后台
         │
         ▼
查看订单列表（状态：待处理）
         │
         ▼
点击"自动下单"
         │
         ▼
系统从账号池获取空闲账号
    ┌────┴────┐
    │ 无可用账号    │ 有可用账号
    ▼               ▼
返回错误       标记账号为使用中
               │
               ▼
启动Puppeteer自动化
               │
               ▼
注入Cookie登录京东
               │
               ▼
打开收货地址页，添加客户地址
               │
               ▼
打开京东家政下单页
               │
               ▼
选择预约时间并提交
    ┌──────────┴──────────┐
    │ 下单失败（重试3次）   │ 下单成功
    ▼                       ▼
记录失败原因            获取京东订单号
更新状态为失败          更新状态为已下单
                        释放账号
```

### 4.3 时间规则匹配流程

```
前端请求 GET /api/time-slots?city=城市名
         │
         ▼
按规则类型匹配（优先级从高到低）：
1. city 类型 → 精确匹配城市名
2. tier 类型 → 在match_value中查找城市
3. global 类型 → 全局默认规则
         │
         ▼
生成从明天起的日期列表
（过滤非工作日，最多max_days天）
         │
         ▼
locked_days 内的日期 → is_full: true（约满）
locked_days 后的日期 → 返回可用时间段
         │
         ▼
返回时间表给前端
```

## 五、非功能性需求

### 5.1 性能要求
- H5页面首屏加载时间 < 3秒
- API接口响应时间 < 500ms
- 支持并发访问（同时100个用户）

### 5.2 安全要求
- 后台接口全部使用JWT鉴权
- 管理员密码使用bcrypt加密（cost factor 10）
- 防止卡密暴力枚举（可考虑接口限流）
- 京东账号密码加密存储

### 5.3 可用性要求
- 系统可用性 > 99%
- 支持宝塔面板PM2进程守护
- 数据库定期备份

### 5.4 兼容性要求
- H5页面支持iOS Safari、Android Chrome
- 后台管理支持Chrome、Firefox、Edge
- 分辨率：H5支持320px-768px，后台支持1280px+
